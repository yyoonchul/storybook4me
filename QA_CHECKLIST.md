# 📋 QA 체크리스트 (사용자 관점)

이 문서는 Storybook4me 애플리케이션의 사용자 관점에서 테스트해야 할 항목들을 정리한 QA 체크리스트입니다.

---

## 🔐 1. 인증 및 계정 관리

### 1.1 회원가입 및 로그인
- [ ] **회원가입**
  - [ ] 이메일/비밀번호로 회원가입 가능
  - [ ] 소셜 로그인 (Google, Apple 등) 작동 확인
  - [ ] 회원가입 후 자동 로그인 확인
  - [ ] 중복 이메일 가입 시도 시 적절한 에러 메시지 표시
  - [ ] 비밀번호 유효성 검사 (최소 길이, 특수문자 등)

- [ ] **로그인**
  - [ ] 정상적인 이메일/비밀번호로 로그인 가능
  - [ ] 잘못된 비밀번호 입력 시 에러 메시지 표시
  - [ ] 존재하지 않는 이메일 입력 시 에러 메시지 표시
  - [ ] 소셜 로그인 정상 작동
  - [ ] 로그인 후 이전 페이지로 리다이렉트 또는 메인 페이지로 이동

- [ ] **로그아웃**
  - [ ] 로그아웃 버튼 클릭 시 세션 종료
  - [ ] 로그아웃 후 보호된 페이지 접근 불가 확인
  - [ ] 로그아웃 후 메인 페이지로 이동

### 1.2 사용자 프로필
- [ ] **프로필 조회** (`/settings/account`)
  - [ ] 로그인한 사용자의 프로필 정보 정상 표시
  - [ ] 프로필 이미지 표시
  - [ ] 이메일, 이름 등 기본 정보 표시
  - [ ] 생성한 스토리북 수, 사용한 크레딧 등 통계 정보 표시

- [ ] **프로필 수정**
  - [ ] 이름 변경 가능
  - [ ] 프로필 이미지 업로드/변경 가능
  - [ ] 변경사항 저장 확인

## 📚 2. 스토리북 생성 및 편집 (Studio)

### 2.1 스토리북 생성
- [ ] **메인 페이지에서 생성**
  - [ ] 메인 페이지의 스토리 입력창에 프롬프트 입력
  - [ ] 화살표 버튼 클릭 시 스토리북 생성
  - [ ] 비로그인 상태에서 생성 시도 시 로그인 모달 표시
  - [ ] 생성 후 `/studio/{id}?mode=settings`로 리다이렉트

- [ ] **Studio 페이지 접근**
  - [ ] 새 스토리북 생성 시 Settings 모드로 시작
  - [ ] 기존 스토리북 편집 시 Preview 모드로 시작
  - [ ] URL 파라미터 `prompt`가 있을 때 프롬프트 기반 생성 플로우 작동

### 2.2 스토리 설정 (Settings 모드)
- [ ] **메인 콘셉트 (Synopsis)**
  - [ ] 프롬프트가 있을 경우 자동으로 입력됨
  - [ ] 메인 콘셉트 텍스트 입력/수정 가능
  - [ ] 첫 생성 시에만 편집 가능 (이미 생성된 스토리는 읽기 전용)

- [ ] **캐릭터 선택 (Characters)**
  - [ ] "Select Characters" 버튼 클릭 시 모달 열림
  - [ ] 내 캐릭터 목록 표시
  - [ ] 프리셋 캐릭터 목록 표시
  - [ ] 캐릭터 선택/해제 가능
  - [ ] 선택한 캐릭터가 하단에 표시됨
  - [ ] "Select All" 기능 작동
  - [ ] "Clear Selection" 기능 작동
  - [ ] 첫 생성 시에만 편집 가능

- [ ] **아트 스타일 선택 (Style)**
  - [ ] 아트 스타일 캐러셀에서 스타일 선택 가능
  - [ ] 좌우 화살표로 스타일 탐색 가능
  - [ ] 선택한 스타일이 하이라이트됨
  - [ ] 첫 생성 시에만 편집 가능

- [ ] **스토리 생성 버튼**
  - [ ] 메인 콘셉트가 입력되어 있을 때만 활성화
  - [ ] "Generate Story" 버튼 클릭 시 생성 시작
  - [ ] 생성 중 로딩 상태 표시
  - [ ] 생성 완료 후 Preview 모드로 전환

### 2.3 스토리 편집 (Preview 모드)
- [ ] **제목 편집**
  - [ ] 상단 제목 입력창에서 제목 수정 가능
  - [ ] 제목 변경 시 자동 저장 (debounce)
  - [ ] 저장 상태 표시 (Saving... / Saved / Save failed)
  - [ ] 저장 실패 시 에러 메시지 표시

- [ ] **페이지 편집**
  - [ ] 각 페이지의 텍스트 내용 편집 가능
  - [ ] 텍스트 변경 시 자동 저장
  - [ ] 저장 상태 표시
  - [ ] 페이지 번호 표시 정확성 확인

- [ ] **페이지 추가/삭제**
  - [ ] "Add Page" 버튼으로 새 페이지 추가 가능
  - [ ] 페이지 추가 후 목록에 반영
  - [ ] 페이지 삭제 버튼 클릭 시 확인 다이얼로그 표시
  - [ ] 삭제 확인 후 페이지 제거
  - [ ] 삭제 후 현재 페이지 인덱스 조정

- [ ] **페이지 네비게이션**
  - [ ] Previous/Next 버튼으로 페이지 이동
  - [ ] 첫 페이지에서 Previous 버튼 비활성화
  - [ ] 마지막 페이지에서 Next 버튼 비활성화
  - [ ] 하단 페이지 인디케이터 클릭으로 이동 (구현 여부 확인)
  - [ ] 페이지 번호 표시 정확성

### 2.4 AI 채팅 편집 (Plus 기능)
- [ ] **무료 플랜 제한**
  - [ ] 무료 플랜 사용자에게 AI 채팅 기능 비활성화 표시
  - [ ] 채팅 입력창 비활성화 및 "Upgrade" 버튼 표시
  - [ ] 채팅 시도 시 업그레이드 안내 메시지 표시

- [ ] **Plus 플랜 기능**
  - [ ] Plus 플랜 사용자는 AI 채팅 사용 가능
  - [ ] 채팅 입력창에 메시지 입력 가능
  - [ ] Enter 키 또는 Send 버튼으로 메시지 전송
  - [ ] AI 응답이 채팅 히스토리에 표시
  - [ ] AI가 스토리를 수정한 경우 변경사항이 Preview에 반영
  - [ ] "Thinking" 상태 표시 (initial/edit/question)
  - [ ] 모든 페이지가 준비되지 않았을 때 적절한 안내 메시지

- [ ] **채팅 히스토리**
  - [ ] 사용자 메시지와 AI 응답이 구분되어 표시
  - [ ] 스크롤 가능한 채팅 영역
  - [ ] 긴 메시지의 경우 줄바꿈 정상 작동

### 2.5 Studio 모드 전환
- [ ] **Preview ↔ Settings 전환**
  - [ ] 상단 토글 버튼으로 모드 전환 가능
  - [ ] Preview 모드에서 스토리북 미리보기 표시
  - [ ] Settings 모드에서 스토리 설정 표시
  - [ ] 모드 전환 시 상태 유지

- [ ] **저장 및 완료**
  - [ ] "Save Storybook" 버튼 클릭 시 `/book/{id}`로 이동
  - [ ] 저장되지 않은 변경사항이 있을 경우 경고 표시 (구현 여부 확인)

   - 개발 환경과 프로덕션 환경 모두에서 테스트
   - 다양한 브라우저와 디바이스에서 테스트

2. **테스트 순서**
   - 인증 → 기본 기능 → 고급 기능 순서로 진행
   - 각 기능을 독립적으로 테스트한 후 통합 테스트

3. **버그 리포트**
   - 발견한 버그는 다음 정보와 함께 기록:
     - 재현 단계
     - 예상 동작
     - 실제 동작
     - 스크린샷/동영상
     - 브라우저/디바이스 정보

4. **우선순위**
   - 🔴 Critical: 앱 사용 불가능한 버그
   - 🟠 High: 주요 기능 작동 안 함
   - 🟡 Medium: 기능은 작동하나 UX 문제
   - 🟢 Low: 미세한 UI/UX 개선 사항
